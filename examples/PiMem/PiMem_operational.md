# PiMem — Operational Document
`v0.1.0` // `operational` // [living]

---

## Target assumption

These rules target a single curated `MEMORY.md` file the agent can both read and overwrite. Extensions that are append-only require human-managed contradiction handling — flag this to the project maintainer if relevant.

---

## Memory categories

Four categories. The agent may only write entries that fit one of these. Anything that doesn't fit is not written.

**Project constants** — stack, key paths, conventions, tooling decisions. True until explicitly changed. Written once; overwritten when superseded, never duplicated.

**Earned lessons** — what failed, what that revealed, what to do differently. Written when a failure or correction has been confirmed, not hypothesized. One entry per distinct failure pattern.

**Active context** — current state: open decisions, in-progress work, known blockers. Short shelf life. Overwritten each session if still relevant; retired when resolved.

**Hard constraints** — boundaries the agent must not cross: never do X, always require Y before Z. Written only on explicit human instruction. Never self-generated by the agent.

---

## Expiry criteria

Entries expire when they meet any of the following conditions. The agent flags candidates; the human retires them.

| Category | Expires when |
|---|---|
| Project constants | Superseded by a new decision in the same session |
| Earned lessons | The failure pattern it describes no longer applies to the current stack or approach |
| Active context | The decision or blocker it describes has been resolved |
| Hard constraints | Explicitly revoked by the human |

**Contradiction rule:** if a new write concerns a fact already recorded, overwrite the existing entry — do not append a second entry. Two entries for the same fact is always a contradiction, never additional context.

---

## Write constraints

**Ceiling:** 20 entries maximum across all categories. At ceiling, no new entry is written without retiring an existing one first. The agent proposes the retirement candidate; the human confirms.

**Per-entry length:** 2 sentences maximum. If it cannot be stated in 2 sentences it is not ready to be a memory entry — flag it for human judgment instead.

**Trigger:** write only at session end, and only if at least one entry in any category has changed, been superseded, or a new confirmed entry exists. A session with no new constants, no failures, no resolved context, and no new constraints produces no memory writes.

**Never write:**
- Speculation or hypotheses not yet confirmed
- Content that requires session context to be understood
- Anything a fresh reader could not act on without asking a clarifying question
- Secrets, credentials, or personally identifying information

---

## Retirement mechanism

The agent does not delete entries unilaterally. When an entry meets expiry criteria, the agent flags it with a one-line note: `[RETIRE?] <reason>`. The human reviews and deletes. This applies at session end during the same pass as new writes.
